## 修改点4 - AI优化版

1. **打组框内节点可拖拽（已修复）**
   打组框所有 DOM 层改为 `pointer-events: none`（纯视觉），不再遮挡节点；框内空白选组通过 `@pane-click` 坐标检测实现；框整体拖动仍由顶部 strip 负责。

2. **新增「脚本节点」（类型名 `script`）**
   - 注册位置与现有节点一致：新建 `src/components/nodes/ScriptNode.vue`，在 `canvas.js` 的 `getDefaultNodeData` switch 中补充 `'script'` case，在 `Canvas.vue` 的 `nodeTypes` 对象中注册，在左侧工具栏 `tools` 数组中添加入口（图标可复用 `DocumentTextOutline`）。
   - 节点 `data` 初始结构：`{ label: '脚本生成器', prompt: '', model: DEFAULT_CHAT_MODEL, taskId: null, status: 'idle', scenes: [], viewMode: 'table' }`
   - **节点布局**（上下两区域）：
     - 上：**正方形预览区**（宽高相等，约 280px）。`status === 'idle'` 时显示占位文案「点击生成分镜脚本」；点击占位区时，调用 `addNode('text', ...)` + `addEdge(...)` 在节点左侧创建文本节点并连接，文本节点预填示例剧本（存为组件内常量 `SAMPLE_SCRIPT`，内容见原始需求），文本节点内容支持用户自行修改或粘贴。
     - 下：**提示词输入区**。多行 textarea，`placeholder` 为「描述剧情或添加角色参考、视频参考等，为你生成分镜脚本」；左下角模型选择器（复用 `useModelStore().availableChatModels`）；右下角执行按钮，`prompt` 为空时 `disabled`。
   - Handle 配置：左侧 `script-input`（接收文本节点输入），右侧 `script-output`（输出到后续节点），与现有节点 Handle 风格一致。

3. **执行流程：异步生成并轮询**
   - 点击执行后，在 `src/api/` 下新增（或复用）接口调用：`POST /script/create` 传入 `{ prompt, model }`，返回 `{ taskId }`；`GET /script/query?taskId=xxx` 返回 `{ status, progress, result }`。
   - 执行中：`updateNode(id, { status: 'loading', taskId })`，正方形区域显示进度条（`<n-progress>` 或 CSS 动画），用 `setInterval(2000)` 轮询，组件卸载时（`onUnmounted`）清除定时器。
   - 完成后：`updateNode(id, { status: 'done', scenes: result })`，`scenes` 为数组，每项包含字段：`{ sceneNo, description, character1, characterDesc1, characterImg1, action, sceneTag, lighting, dialogue, storyboardPrompt, videoMotionPrompt }`。
   - 失败时：`updateNode(id, { status: 'error' })`，显示错误提示与重试按钮。

4. **脚本表格交互**（`status === 'done'` 时显示）
   - **全屏 / 恢复**：右上角图标切换，全屏用 `position: fixed; inset: 0; z-index: 9999`；恢复后回到节点内正常布局。
   - **镜号拖拽排序**：每行首列可拖拽，用 `dragstart` / `dragover` / `drop` 在 `data.scenes` 数组中做位置交换，拖拽后调用 `updateNode` 持久化。
   - **视图切换**：工具栏提供「脚本视图」（完整表格）和「创意视图」（每镜一卡片，展示镜号 + 画面描述 + 分镜提示词）；切换仅改变 `data.viewMode`，参考图 `6-分镜脚本UI.png`。

5. **脚本表格工具栏**
   - **重新生成**：清空 `data.scenes`、重置 `status: 'idle'`，再次触发步骤 3 执行流程。
   - **生成分镜**：遍历 `data.scenes`，为每行的 `storyboardPrompt` 调用 `addNode('imageConfig', pos, { prompt, label: '分镜#N' })` 批量创建图片配置节点（用 `addNodes` 批量写入），然后调用 `addCanvasGroup(newNodeIds)` 将所有新节点打成一个组，组标题默认为「分镜图 · 脚本生成器」（双击可改名，与现有打组 `label` 逻辑一致）；组内每个 `imageConfig` 节点保留独立「重新生成」按钮，参考图 【7-生成分镜UI.png】。
