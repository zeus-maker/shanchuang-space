# 打组框错位：与节点不在同一变换层

**日期：** 2025-03-26

## 背景与原因

- **现象：** 框选后点「打组」，半透明边框出现在画布一侧（如最左侧），与真实节点分离。
- **根因：** 分组框使用与节点相同的**流坐标**（`position.x/y` 等），但渲染在了 Vue Flow **默认插槽**里。在该版本中，默认插槽内容在**视口 `translate + scale` 变换之外**，坐标不会被平移/缩放，相当于把「流空间里的数字」当成了外层容器的像素，导致与节点视觉脱节。

## 修改点（行为层面）

- 分组框与分组工具栏改到与节点、边同一变换层（通过 **`#zoom-pane` 插槽**），框再次随缩放与平移与成员对齐。

## 复盘

- 自定义覆盖层凡是使用**流坐标**定位，都必须与节点同层；若用屏幕坐标则需自行 `flowToScreenCoordinate` 并随视口更新，成本更高。
