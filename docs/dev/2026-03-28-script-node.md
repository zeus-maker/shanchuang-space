# 脚本生成器节点（ScriptNode）

**日期：** 2025-03-26

## 背景与原因

- **需求来源：** `docs/sprint/1-change.md`「修改点4」——用户希望在画布上通过 LLM 自动生成分镜脚本，并能一键将脚本中每条分镜转换为可生成图片的节点组。
- **与旧方案的差异：** 旧流程需要先用 LLMConfigNode 手动生成文本，再人工拆解成图片节点。ScriptNode 把「生成脚本 → 编辑脚本 → 批量出图」三步合并到一个节点内，降低操作门槛。

## 设计与行为说明（用户可见）

### 状态一：空置（idle）
节点显示一个方形占位区域（286×286），内有列表图标。
- **点击占位区域**：若当前节点没有输入连接，自动在左侧创建一个「文本节点」并预填《我在盛唐写天下》示例剧本，通过连线接入。方便用户直接体验整个流程。
- **底部输入框**：填写生成要求（如角色参考、视频参考等）。placeholder："描述剧情或添加角色参考、视频参考等，为你生成分镜脚本"。
- **模型选择**：底部左侧可选取对话模型（默认 deepseek-chat）。
- **执行按钮**：须至少有一个文本输入（连接的文本节点内容 或 底部提示词），否则禁用。

### 状态二：生成中（loading）
占位区变为进度条 + 已生成字数提示，实时反映流式输出进度。

### 状态三：完成（done）
节点宽度自动扩展至 960px，展示工具栏 + 分镜表格。

**表格列：** 镜号 / 时长 / 画面描述 / 角色1 / 角色描述1 / 角色图1 / 角色动作 / 场景标签 / 光影氛围 / 对白 / 分镜提示词 / 视频运动提示词

- **行拖拽排序**：任意拖动行即可重新排序，镜号自动重新编号。
- **脚本视图 / 创意视图**：顶部右侧可切换，创意视图用 4 列卡片展示每个分镜的图片占位 + 描述。
- **全屏展开（↗）**：使用 `Teleport to body` 覆盖整个屏幕，适合大量分镜的浏览与编辑。

**工具栏按钮：**
- **重新生成**：回到空置状态，可修改提示词后重新调用。
- **生成分镜**：按 5 列网格批量创建 `imageConfig` 节点（每条分镜的 `storyboardPrompt` 作为生图提示词），并自动打组，组名为"分镜图 · 脚本生成器"。

## 技术决策

- **LLM 调用方式**：复用项目已有的 `streamChatCompletions`（流式调用），通过 System Prompt 约定模型直接输出 JSON 数组，无需单独后端任务队列接口。
- **JSON 鲁棒解析**：`parseScriptJSON` 依次尝试直接解析 / 提取 markdown 代码块 / 提取首个 `[...]` 区段，兼容模型输出差异。
- **节点宽度动态切换**：通过 `:style="{ width: status === 'done' ? '960px' : '340px' }"` 并在 `status` 变化后调用 `updateNodeInternals(id)` 通知 Vue Flow 重新计算布局。
- **打组延迟**：`生成分镜` 执行时先批量 `addNodes`，再等待 120ms 后调用 `addCanvasGroup`，确保节点已写入响应式数组并能被 `computeGroupBounds` 正确测量。
- **组自定义 label**：`addCanvasGroup` 返回新组 ID 后立即调用 `updateCanvasGroup(id, { label: '...' })`，Canvas.vue 的组标题改为 `g.label || 默认文案`。

## 涉及模块

- `src/components/nodes/ScriptNode.vue`（新增）
- `src/stores/canvas.js`：`estimateNodeSize` 新增 `script: [340, 450]`；`getDefaultNodeData` 新增 `script` 分支
- `src/views/Canvas.vue`：注册节点类型、工具栏入口、打组标题支持 `label`

## 复盘

- **AI 生成 JSON 的稳定性**：System Prompt 要求直接输出数组而不加代码块，但实际模型有时仍会包裹 ` ```json ``` `，所以保留多级解析是必要的。
- **节点宽度与 Vue Flow**：动态宽度变化后若不调用 `updateNodeInternals`，边（edge）不会随之重新贴附到节点边缘，需要留意。
- **生成分镜的等待时间**：120ms 延迟是经验值，若在高负载场景节点未及时入队可能导致组边框偏小，后续可改为监听 `nodes` 写入完成事件（如 `nextTick` + `requestAnimationFrame`）替代固定延迟。
