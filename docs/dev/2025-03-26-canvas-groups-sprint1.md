# 画布 Sprint1：悬停平移与区域打组

**日期：** 2025-03-26

## 背景与原因

- **需求来源：** `docs/sprint/1-change.md`「修改1」——希望画布操作更直觉：平移不依赖「按住空格再拖」；打组是「圈一块区域」而不是多出一个逻辑节点。
- **与旧方案的差异：** 此前打组会新建「组引用」文本节点，下游靠连线读 `bundleMemberIds` 聚合内容。新方案把「组」当成**画布上的视觉分组**：成员仍是原有节点，分组只描述「谁在同一框里」及框的样式，避免多一个节点类型干扰拓扑与心智模型。
- **平移的取舍：** 在**空白画布**上、鼠标未按键时，用指针位移带动视口平移；节点上的交互仍由节点接管，左键在空白处拖框多选保持 Vue Flow 默认行为。这样满足「移动即可平移」，又尽量不抢走框选与节点拖拽。

## 设计与行为说明（用户可见）

1. **平移**  
   焦点不在输入类控件时，在画布空白区域移动鼠标即可平移视图（跟手位移，无按键）。

2. **打组**  
   多选至少两个节点后点「打组」，出现**半透明、虚线边框**的矩形区域，随成员位置变化而**自动包住**成员（不是手动画固定矩形）。刷新或重进项目后分组仍在。

3. **点击分组区域**  
   出现工具栏：**底色**（若干预设 + 不使用填充）、**排列**（宫格 / 水平，重排成员位置）、**整组执行**（弹窗展示按连线推演的先后顺序，并说明自动串联生成尚未接好，避免误以为已全自动）、**添加到工具箱**（写入浏览器本地，便于以后扩展「工具箱面板」）、**解组**（只取消分组与框，不删节点）、**批量下载**（组内带链接的图片/视频）。

4. **删除与兼容**  
   若成员节点被删，分组会自动收缩成员列表；不足两人则分组消失。旧数据里若仍存在「组引用」文本节点，仍可按原逻辑被下游解析，与新分组方式可并存。

## 涉及模块（便于代码对照，非实现说明书）

画布页与画布状态：`Canvas.vue`、`canvas.js`（分组列表随项目持久化，并与撤销/重做快照一致）。

## 复盘

- **文档写法：** 初版留档偏「函数与字段罗列」，对读文档的人（产品、后续自己）不友好；应以**为何这样做、用户看到什么、哪些故意没做**为主，代码锚点只作索引。
- **未完项：** 整组按顺序自动调用各节点生成能力仍待接入；工具箱目前仅本地存储，无统一 UI 入口。
