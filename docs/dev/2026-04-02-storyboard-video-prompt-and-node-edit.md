# 分镜批量视频：运动提示词传递与视频节点可编辑

## 背景与原因

1. **图生视频提示词**：从分镜图批量生成分镜视频时，依赖「脚本生成器」表格里的「视频运动提示词」；仅靠组内下标与 `scenes[i]` 对齐，在镜号与节点顺序不一致、或画布侧无法从打组代理解析到脚本节点时，容易拿不到 `videoMotionPrompt`，退化成空或仅用画面描述。另：由脚本一键创建的分镜图节点未把 `videoMotionPrompt` 写入节点数据，断链后无法兜底。
2. **视频节点编辑**：批量生成的分镜视频节点在生成完成后，需要能查看/修改图生视频用的提示词，并在保留首帧（及尾帧）参数的前提下支持重新发起任务；此前轮询成功还会把节点标题统一改成「视频生成」，破坏「分镜视频 #N」标签。

## 修改点

- 新建 `storyboardVideoPrompt.js`：按节点上的 `sceneNo` 或标题「分镜 #N」匹配脚本行，**优先**使用节点或脚本中的 `videoMotionPrompt`，再回退到画面描述 / 分镜图 `prompt`。
- 脚本「生成分镜图」创建的 `imageConfig` 节点写入 `sceneNo`、`videoMotionPrompt`，与脚本表格一致。
- `ScriptNode` / `Canvas` 批量生视频均改用上述解析逻辑；画布侧 `findScriptScenesForGroup` 增加对 `script-output` 直连组内节点或组代理的补充解析。
- 批量创建的视频节点写入 `videoMotionPrompt` 与 `videoGenParams`（模型、比例、时长、分辨率、配音、首/尾帧图），供节点内编辑与重生成。
- `VideoNode`：在已生成完成且具备批量元数据时展示「图生视频提示词」多行输入（失焦保存）、可选「用当前提示词重新生成」；轮询成功时保留原有 `label`。

## 复盘

- 旧画布中未带 `videoMotionPrompt` 的分镜图节点仍可通过标题镜号 + 脚本草稿对齐；若既无镜号又无脚本关联，则只能依赖分镜图上的 `prompt`。
- 纯本地上传、无 `videoGenParams` 的视频节点不展示该编辑区，避免与「分镜批量视频」场景混淆。
