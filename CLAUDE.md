## 日期规则（最高优先级）

**`docs/dev/` 文件命名的日期必须是系统实际日期，不得使用 `user_info` 元数据中的 "Today's date" 字段。**

原因：`user_info.Today's date` 可能与系统时钟不一致，已多次导致文档日期错误。

正确做法：**每次创建 `docs/dev/` 文件前，必须先执行 `date +%Y-%m-%d` 获取真实日期，并以该输出作为文件名前缀。**

```bash
# 每次创建 docs/dev 文件前必须先运行：
date +%Y-%m-%d
# → 以输出结果（如 2026-03-28）命名文件
```

## docs/dev 留档规则

- 每次完成 Bug 修复、功能实现或配置调整后，必须在 `docs/dev/` 下新建留档文件
- 命名格式：`YYYY-MM-DD-<简短英文-slug>.md`（日期来自 `date` 命令）
- 内容结构：**背景与原因 → 修改点 → 复盘**，侧重"为什么"而非函数堆砌
- 留档与代码修改宜在同一个 commit 中提交
